# 1.0.0 Export-Surface Audit — Findings

Export-surface audit for 1.0.0 (3 read-only auditors across 15 modules).

## Removed from public API

| Symbol | Module | Reason |
|---|---|---|
| `TENANT_SCOPED_METADATA` | `orm` | Reflect metadata key; internal implementation detail |
| `TENANT_SCOPED_COLUMNS` | `orm` | Reflect metadata key; internal implementation detail |
| `SEARCHABLE_FIELDS_METADATA` | `orm` | Reflect metadata key; internal implementation detail |
| `SCHEMA_METADATA` | `orm` | Reflect metadata key; internal implementation detail |
| `SCHEMA_PROP_METADATA` | `orm` | Reflect metadata key; internal implementation detail |
| `AUDIT_ENABLED_METADATA` | `orm` | Reflect metadata key; internal mixin implementation detail |
| `getHistoryRecorder` | `orm` | Internal accessor used by `base-repository.ts`; not part of public extension API |
| `DEFAULT_HEADERS_CONFIG` | `context` | Default config constant; consumers configure via `ContextModule.forRoot()` options |
| `CacheEntry` | `cache` | Internal cache storage shape; consumers interact via `CacheService`, not the raw entry type |
| `toIssues` | `validation` | Internal helper called by `ZodValidationGuard`; not part of consumer validation API |
| `mergeCatalogs` | `i18n` | Internal catalog merging utility; consumers pass catalogs through `I18nModule.forRoot()` options |
| `slugify`, `isBlank`, `toMb`, `addDays`, `distinct` | `file-storage` | Internal string/date utilities leaked via barrel (addressed in earlier task) |
| tenancy decorator re-export + `getScopedColumns` leak | `tenancy` / `orm` | Duplicate tenancy decorator re-export and getScopedColumns barrel leak (addressed in earlier task) |

## Added for Completeness

`INTERNAL_CONTEXT_ENRICHER` + `IInternalContextEnricher` are now re-exported from the root index (`src/index.ts`). These are the documented DI extension point for enriching trusted internal request context — they were missing from the public surface despite being a first-class consumer extension point documented in the source file.

## Kept (uncertain, conservative)

The following symbols were judged as plausible extension points and left public for 1.0.0. They can be narrowed in a future semver-minor audit if usage evidence shows they are purely internal.

| Symbol | Note |
|---|---|
| `ClassRef`, `Constructor` | Generic type utilities; potentially useful to consumers extending base classes |
| `isAuditEnabled` | Mixin introspection; useful for consumers conditionally applying audit behaviour |
| `getScopedColumns` | Column introspection; useful for consumers building multi-tenant query helpers |
| `getSearchableConfig` | Config introspection; useful for consumers building search UIs |
| `getSchema`, `getSchemaProps` | Schema introspection; useful for consumers building serialization/validation |
| `ContextMiddleware` | NestJS middleware; consumers may need to apply it explicitly in AppModule |
| `RequestCacheMiddleware` | NestJS middleware; consumers may need to apply it explicitly in AppModule |
| `CACHED_METADATA` | Decorator metadata key; may be needed by consumers inspecting cache config |
| `DEFAULT_JOB_OPTIONS`, `DEFAULT_QUEUE_PREFIX` | Job scheduler defaults; consumers may override selectively |
| `SimpleI18nResolver`, `SimpleI18nOptions` | Concrete i18n resolver; consumers may register or extend it |
| `DefaultAuditStrategy` | Concrete strategy; consumers may extend it |
| `DefaultPermissionStrategy` | Concrete strategy; consumers may extend it |
| `DefaultLanguageResolver` | Concrete resolver; consumers may extend it |
| `ACTION_HISTORY_ACTOR_RESOLVER` | DI token for action-history actor; consumers provide implementations |

Post-audit: stateful modules grouped under `src/features/` and exposed via a single `@sdcorejs/nestjs/features` entry; the interim central `entities` entry + `SD_CORE_ENTITIES` were removed.
