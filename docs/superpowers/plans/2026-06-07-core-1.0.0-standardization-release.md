# @sdcorejs/nestjs 1.0.0 Standardization + Release Automation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Standardize the public API of `@sdcorejs/nestjs` (clean break), wire Changesets-based publish-on-merge-to-main, and ship `1.0.0` to npm.

**Architecture:** Refactor-and-release. Remove duplicate/aliased/leaking exports so every public symbol has one canonical sub-path, then validate the package surface with `publint` + `attw`, then replace the tag-based release workflow with `changesets/action`. The only consumer (`enterprise-platform`) is vendored and out of scope.

**Tech Stack:** TypeScript 5.6, NestJS 11, tsup (dual ESM/CJS), tsc (`.d.ts`), Jest + ts-jest, Changesets, GitHub Actions, publint, @arethetypeswrong/cli.

**Spec:** [docs/superpowers/specs/2026-06-07-core-1.0.0-standardization-release-design.md](../specs/2026-06-07-core-1.0.0-standardization-release-design.md)

**Branch:** work on `release/0.1.0` (current branch of this repo). Each task commits separately.

**Pre-existing uncommitted work:** the earlier `src/utils/` removal + `orm/index.ts`/docs edits are uncommitted on this branch. Commit or stash them before starting (Task 0).

Replace the entire contents of `.github/workflows/release.yml` with:

```yaml
name: Release
on:
  push:
    branches: [main]
concurrency: ${{ github.workflow }}-${{ github.ref }}
jobs:
  release:
    name: Version or publish
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
      id-token: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20.x'
          registry-url: 'https://registry.npmjs.org'
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Verify (lint + test + build + export check)
        run: |
          npm run lint
          npm run test:coverage
          npm run build
          npm run check:exports

      - name: Create Release PR or publish
        uses: changesets/action@v1
        with:
          version: npm run changeset:version
          publish: npm run release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## Notes on TDD scope

- **Value-level export removals** (decorators, functions like `distinct`) get runtime guard tests (`*.public-api.spec.ts`) — the test fails before the fix, passes after.
- **Type-only removals** (`Sd*` aliases, type re-exports) are erased at runtime, so they are verified by `npm run build:types` (tsc) + `attw` rather than a runtime assertion.
- **Config/CI changes** (workflow, package.json, changeset) are verified by parse/dry-run commands, not unit tests.
